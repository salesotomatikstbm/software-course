JavaScript Basics Quiz

1. Where is the correct place to insert JavaScript in an HTML document?

Both in the <head> and <body> are acceptable
In the <head> section
In the <body> section
In an external .css file

JavaScript can be placed:

  • In the <head> (executes before page renders)
  • In the <body> (executes as it's encountered)
  • In external .js files (recommended for larger projects)
<!-- In head --> <head> <script src="script.js"></script> </head> <!-- Before closing body --> <body> <script src="script.js"></script> </body>

2. Which of these is NOT a way to display output in JavaScript?

console.print()
document.write()
alert()
console.log()

Correct output methods:

  • console.log() - Outputs to browser console
  • document.write() - Writes directly to HTML (rarely used)
  • alert() - Shows popup dialog
  • innerHTML/textContent - Modifies HTML elements

console.print() doesn't exist in standard JavaScript.

3. What is the correct way to declare a variable in modern JavaScript?

let x = 5;
variable x = 5;
v x = 5;
x = 5;

Modern JavaScript has three ways to declare variables:

// Block-scoped variable (can be reassigned) let x = 5; // Block-scoped constant (cannot be reassigned) const y = 10; // Function-scoped or global variable (avoid in modern JS) var z = 15;

let and const are preferred over var in modern JavaScript.

4. Which of these is NOT a JavaScript primitive data type?

array
string
boolean
number

JavaScript primitive data types:

  • string (e.g. "hello")
  • number (e.g. 42, 3.14)
  • boolean (true/false)
  • null
  • undefined
  • symbol (ES6+)
  • bigint (ES2020+)

Array is an object type, not a primitive.

5. What will typeof null return in JavaScript?

"object"
"null"
"undefined"
"number"

This is a well-known quirk in JavaScript:

typeof null; // Returns "object" (historical bug) typeof undefined; // Returns "undefined"

This behavior is from early JavaScript and can't be fixed without breaking existing code.

6. What is the correct syntax for a JavaScript function declaration?

function myFunc() { }
func myFunc() { }
myFunc = function() { }
def myFunc() { }

JavaScript has several ways to define functions:

// Function declaration (hoisted) function myFunc() { } // Function expression const myFunc = function() { }; // Arrow function (ES6+) const myFunc = () => { };

The declaration syntax starts with the function keyword.

7. How can you access all arguments passed to a function, regardless of parameters?

Using the arguments object
Using the params array
Using the ...rest parameter
Both a and c

Two ways to handle variable arguments:

// Traditional arguments object (array-like) function sum() { let total = 0; for (let i = 0; i < arguments.length; i++) { total += arguments[i]; } return total; } // Modern rest parameters (ES6+) function sum(...numbers) { return numbers.reduce((a, b) => a + b, 0); }

The rest parameter syntax is preferred in modern JavaScript.

8. What is a key difference between arrow functions and regular functions?

Arrow functions don't have their own 'this'
Arrow functions can't return values
Arrow functions must have parameters
Arrow functions can't be assigned to variables

Arrow functions (ES6+) differ from regular functions in several ways:

  • No own this (inherits from surrounding scope)
  • No arguments object (use rest parameters instead)
  • Can't be used as constructors (no new)
  • Shorter syntax (often one-line)
// Regular function const obj = { value: 42, print: function() { console.log(this.value); // 42 } }; // Arrow function (inherits 'this') const obj = { value: 42, print: () => { console.log(this.value); // undefined } };

9. What will this code output?
let x = 10;
function test() {
  console.log(x);
  let x = 20;
}
test();

ReferenceError
10
20
undefined

This demonstrates the "temporal dead zone" with let and const:

let x = 10; function test() { // TDZ for x starts here console.log(x); // ReferenceError let x = 20; // TDZ ends here } test();

Unlike var, let declarations aren't hoisted to the top of their scope.

10. What is hoisting in JavaScript?

The process of moving declarations to the top of their scope
A way to optimize code execution
A method for importing modules
A technique for handling asynchronous code

Hoisting is JavaScript's behavior of moving declarations to the top of their scope before execution:

// This works due to hoisting console.log(x); // undefined var x = 5; // Function declarations are also hoisted foo(); // "Hello" function foo() { console.log("Hello"); }

Note: Only declarations are hoisted, not initializations. let and const declarations are hoisted but not initialized.

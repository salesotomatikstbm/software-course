<nav>

3. What's the common approach for mobile navigation on large websites?

Hamburger menu that expands when clicked
Showing all navigation items in tiny font
Requiring users to rotate their device
Using only footer navigation

Hamburger menu implementation:

/* Mobile first */
.nav-menu { display: none; }
.hamburger { display: block; }

/* Desktop */
@media (min-width: 768px) {
  .nav-menu { display: flex; }
  .hamburger { display: none; }
}

nav {

5. Which CSS technique is best for creating a responsive card grid (Pattern 01)?

CSS Grid with auto-fill and minmax()
Floats with clearfix
Absolute positioning
Table layout

CSS Grid for card layouts:

.card-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(250px, 1fr));
  gap: 20px;
}

6. For a sidebar + main content layout (Pattern 02), which approach is most flexible?

CSS Grid with grid-template-columns: 1fr 3fr;
Two floated divs
Absolute positioned sidebar
Flexbox with fixed widths

Sidebar layout with CSS Grid:

.layout {
  display: grid;
  grid-template-columns: 1fr 3fr;
  gap: 20px;
}

@media (max-width: 768px) {
  .layout { grid-template-columns: 1fr; }
}

7. How would you implement a responsive masonry layout (Pattern 03)?

CSS Grid with auto-rows and dense packing
Multiple floated columns
JavaScript calculation
CSS columns property

Modern masonry with CSS Grid:

.masonry {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
  grid-auto-rows: 10px;
  grid-auto-flow: dense;
  gap: 15px;
}
.item {
  grid-row-end: span var(--row-span);
}

8. What's the best way to toggle mobile navigation visibility?

JavaScript to toggle a CSS class
CSS :hover pseudo-class
HTML5 details/summary elements
CSS animations only

Mobile navigation toggle implementation:

// JavaScript
document.querySelector('.hamburger').addEventListener('click', () => {
  document.querySelector('.nav-menu').classList.toggle('active');
});

/* CSS */
.nav-menu {
  display: none;
  /* other styles */
}
.nav-menu.active {
  display: block;
}

9. What's an important accessibility consideration for navigation menus?

Keyboard navigability and ARIA labels
Using only icons without text
Auto-playing navigation instructions
High contrast only for dropdowns

Accessible navigation includes:

  • Keyboard focus styles
  • ARIA attributes (aria-expanded, aria-label)
  • Skip navigation links
  • Proper heading structure
<nav aria-label="Main navigation">
  <ul>
    <li><a href="/" aria-current="page">Home</a></li>
    <li>
      <button aria-expanded="false" aria-controls="submenu">Products</button>
      <ul id="submenu" hidden>...</ul>
    </li>
  </ul>
</nav>

10. When choosing between layout patterns, what's the most important factor?

Content structure and user needs
Latest CSS trend popularity
Browser support for IE11
Developer preference

Choosing layout patterns should consider:

  • Content type: Is it hierarchical? Linear? Modular?
  • User goals: What are users trying to accomplish?
  • Device variety: How will it work across screen sizes?
  • Performance: How complex is the layout?

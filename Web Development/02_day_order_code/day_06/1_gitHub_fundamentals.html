Understanding version control and how to use GitHub

Version Control Basics

Imagine working on a school project with friends. Everyone is editing the same document. It gets messy, right? Version control helps track every change, so you can go back in time, see who did what, and avoid mistakes.

Think of it like:
  • A time machine for your code
  • A shared notebook where each change is saved and labeled
  • You can go back to any earlier version if something breaks

GitHub Setup

GitHub is like a digital locker where you keep all your project versions safe and share them with others. It works with Git to store your files online.

Steps to Get Started:
  1. Go to github.com and create a free account
  2. Download Git from git-scm.com
  3. Open a terminal or Git Bash
  4. Set your name and email:
    5. git config --global user.name "Your Name" git config --global user.email "you@example.com"
  5. Create a new folder for your project
    6. git init
  6. Create a new GitHub repository and link it:
    7. git remote add origin https://github.com/your-username/your-repo.git
Git is your journal. GitHub is your cloud diary, accessible from anywhere.
